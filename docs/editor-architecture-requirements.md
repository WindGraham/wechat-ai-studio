# 微信文章编辑器架构需求报告

## 一、项目概述

本项目是一个基于浏览器的微信文章可视化编辑器，核心目标是将**文档流编辑**、**自由拖动编辑**、**AI编辑**、**微信HTML**四种工作方式统一在同一套底层代码上，实现四位一体的无缝切换。

编辑器以秀米HTML（与微信HTML兼容的格式）为导入基准，在编辑器内部使用Workbench HTML规范作为统一数据结构，支持用户在任意时刻从一种编辑模式切换到另一种，而不丢失内容或排版意图。

---

## 二、现状分析

### 2.1 已有能力

| 能力 | 状态 | 说明 |
|------|------|------|
| 自由画布拖拽 | ✅ 已实现 | 基于interact.js，支持绝对定位块的拖动与缩放 |
| 建组/解组 | ✅ 已实现 | 支持group和flex-row两种容器类型 |
| 导入秀米HTML | ✅ 已实现 | 通过style推断组件类型，转为canvas-block |
| 导出微信HTML | ✅ 已实现 | 清洗编辑器属性，输出inline-style纯HTML |
| Workbench HTML规范 | ⚠️ 部分实现 | 规范已定义，但编辑器未完全遵循 |
| 嵌套group交互降级 | ✅ 已实现 | group内子元素不可单独拖动 |

### 2.2 关键缺失

| 缺失 | 影响 |
|------|------|
| 无"完全解组"功能 | 用户无法将group彻底打散为独立基础单位 |
| 无文档流编辑模式 | 编辑器只有自由画布模式，无自上而下文档流排版 |
| 无模式切换能力 | 自由拖动与文档流之间无法双向转换 |
| 无行列容器动态操作 | 无法动态添加/删除行列，无法在容器内嵌套容器 |
| 无round-trip闭环 | 编辑器导出后无法被自身重新导入 |
| 无独立元件精细调控 | 无法调整字距、行距、首行缩进、图片内外距等 |
| 无全局格式调控面板 | 无法通过边距、行距等参数调整整篇文章版式 |

## 三、目标架构：四位一体

四种编辑方式共享同一套Workbench HTML底层，任意时刻可无损切换。

```
┌─────────────────────────────────────────────────────────────┐
│                      统一底层：Workbench HTML                  │
│  <section data-wb-root="1">                                  │
│    <section data-wb-type="title|text|image|flex-row|...">    │
│  </section>                                                  │
└─────────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        │                     │                     │
        ▼                     ▼                     ▼
   ┌─────────┐        ┌─────────────┐        ┌──────────┐
   │ 文档流  │        │ 自由拖动    │        │ AI 编辑  │
   │ 编辑模式 │  ←→   │ 编辑模式    │  ←→    │ 模式     │
   └─────────┘        └─────────────┘        └──────────┘
        │                     │                     │
        └─────────────────────┴─────────────────────┘
                              │
                              ▼
                    ┌─────────────────┐
                    │  微信 HTML 导出   │
                    │  (秀米兼容格式)   │
                    └─────────────────┘
```

### 3.1 文档流编辑模式

- 组件按声明顺序自上而下垂直排列
- 支持行列容器：flex-row（水平行）、flex-col（垂直列）
- 行和列可动态增删：↔ 左右加一列、后面加一行
- 行是自由的，可在行内放置任意组件
- 容器支持嵌套：容器内可以加容器
- 支持全局格式调控：边距、行距、字距、段间距等
- 组件在容器内自动固定位置，不可自由拖动
- 支持恢复行列容器的格式（从自由拖动模式切回时重组）

### 3.2 自由拖动编辑模式

- 所有基础单位（文本块、图片块、标题块等）完全独立
- 支持绝对定位自由拖动
- 支持缩放（保持比例或自由缩放）
- **完全解组**：一键将所有group/flex-row递归打散，所有叶子节点直接暴露在Canvas上
- 自由拖动模式下**不需要**组件排版的数值调控（如内外边距、行距等）
- 组件之间无容器约束，各自独立定位
- 支持多选、批量对齐、等距分布

### 3.3 AI编辑模式

- AI可直接读取Workbench HTML进行分析和调整
- AI可识别所有`data-wb-type`语义标记
- AI调整后的HTML可被编辑器重新导入，恢复为可编辑状态
- AI可在文档流模式和自由拖动模式之间做智能转换
- AI可基于用户自然语言指令调整整篇文章的排版（如"整体紧凑一点"、"标题加大"等）

### 3.4 微信HTML

- 最终输出格式，与秀米HTML兼容
- 纯inline-style，无class、无id、无data-*属性
- 支持微信编辑器直接粘贴
- 支持Auto-Publish API直接发布
- 从任意模式导出结果一致

## 四、核心功能需求

### 4.1 完全解组（Deep Ungroup）

- 递归打散所有group/flex-row容器，无论嵌套多深
- 最终Canvas上只保留叶子节点（基础单位）
- 每个叶子节点恢复为顶层可拖动状态
- 解组后各元素坐标计算正确（绝对坐标 = 所有父容器偏移叠加）
- 解组后各元素内部`<section>`内容完整保留

### 4.2 基础单位的自由操作

**可视为基础单位的类型：**
- text（正文段落）
- title（标题）
- image（单图）
- divider（分割线）
- badge（标签/徽章）
- spacer（间距）

**每个基础单位支持：**
- 自由拖动（任意位置）
- 缩放（等比或自由）
- 选中（单选/多选/框选）
- 删除
- 复制/粘贴/克隆
- 双击编辑内容

### 4.3 行列容器操作

**flex-row（水平行容器）：**
- 可在左侧或右侧添加一列
- 可删除某一列
- 列宽自动均分或手动调整
- 列内可放置任意基础单位
- 支持列的拖动排序

**flex-col/自由行（垂直列/行容器）：**
- 可在后面添加一行
- 每行是独立的排版单元
- 行内可放置任意组件（包括基础单位和容器）
- 行高自动或手动调整
- 支持行的拖动排序

**容器嵌套：**
- 容器内可以加基础单位
- 容器内可以加其他容器（多层嵌套）
- 嵌套层级无限制

### 4.4 文档流模式下的全局格式调控

**整篇文章级：**
- 全局段前距/段后距
- 全局行距倍数
- 全局字距（letter-spacing）
- 页面左右边距
- 组件间默认间距

**容器级：**
- 容器内边距（padding）
- 容器外边距（margin）
- 容器内组件间距（gap）
- 容器背景色
- 容器圆角

### 4.5 独立元件精细调控

**文本类元件（text/title）：**
- 字号
- 字重
- 字色
- 字距（letter-spacing）
- 行高（line-height）
- 首行缩进（text-indent）
- 对齐方式（左/中/右/两端）
- 段前距/段后距

**图片类元件（image）：**
- 宽度（百分比或固定值）
- 圆角
- 外边框
- 阴影
- 上边距/下边距
- 水平对齐

**容器类元件（card/flex-row/flex-col）：**
- 背景色
- 边框
- 圆角
- 内边距（四向独立或统一）
- 外边距（四向独立或统一）

### 4.6 模式切换

**自由拖动 → 文档流：**
- 所有绝对定位组件按Y坐标排序
- 相邻组件自动组合为行或列（根据X坐标重叠度判断）
- 或按用户选择的区域批量转换为容器
- 组件进入容器后失去绝对定位，变为文档流排列

**文档流 → 自由拖动：**
- 完全解组（可选：保留容器结构或完全打散）
- 所有组件转为绝对定位
- 按当前文档流位置计算绝对坐标
- 恢复自由拖动能力

**切换时的状态保持：**
- 内容不丢失
- 样式尽可能保留
- 坐标计算精确

## 五、导入导出与闭环

### 5.1 导入

**支持导入的格式：**
- 秀米HTML（原始模板）
- 微信HTML（任意来源）
- Workbench HTML（编辑器自身导出）

**导入时的类型识别：**
- 有`data-wb-type`时直接识别
- 无`data-wb-type`时通过style推断（兼容现有秀米导入逻辑）

**导入后的初始状态：**
- 默认进入文档流模式或自由拖动模式（根据导入内容判断）
- 或让用户选择进入哪种模式

### 5.2 导出

**支持导出的格式：**
- 微信HTML（最终交付格式）
- Workbench HTML（AI编辑/round-trip格式）

**微信HTML导出规则：**
- 移除所有`data-wb-*`属性
- 移除`.canvas-block`包装层
- 移除`position: absolute`（转为文档流或负margin层叠）
- 保留所有inline style
- `flex-row`转为`inline-block`行容器
- `group`完全展开，去除包装

**Workbench HTML导出规则：**
- 保留`data-wb-type`语义标记
- 保留`data-wb-root="1"`
- 保留容器结构（flex-row/flex-col/group）
- 保留所有style
- 可被编辑器重新导入

### 5.3 Round-trip 闭环

```
编辑器 → 导出 Workbench HTML
      → AI 分析并调整
      → 导入编辑器（恢复可编辑状态）
      → 用户继续编辑
      → 导出微信HTML（最终交付）
```

## 六、用户体验目标

| 场景 | 体验目标 |
|------|---------|
| 导入秀米模板 | 一键导入，自动识别所有组件类型，正确还原排版 |
| 完全解组 | 一键打散所有容器，所有基础单位立即可自由拖动 |
| 自由拖动排版 | 像PPT一样自由摆放，不需要关心边距数值 |
| 切换到文档流 | 组件自动排列整齐，恢复行列容器，可精细调控 |
| 调整整篇文章 | 修改一个全局参数（如行距），全文同步变化 |
| 调整单个图片 | 选中图片，直接调宽度、圆角、边距 |
| 调整一段文字 | 双击进入编辑，可调整字距、行距、缩进等 |
| AI辅助排版 | 告诉AI"紧凑一点"，AI自动调整全文参数 |
| 导出到微信 | 一键导出，直接粘贴到公众号编辑器 |

## 七、边界与约束

- 编辑器画布宽度固定375px（微信移动端标准宽度）
- 微信HTML最终输出不支持`position: absolute`（导出时转换）
- 微信HTML不支持`animation`、`transition`、`display: grid`
- SVG内容有条件支持（需单独验证）
- 所有图片URL最终必须为公网HTTPS或微信CDN
- 自由拖动模式下的重叠排版，导出为文档流时可能产生负margin层叠
